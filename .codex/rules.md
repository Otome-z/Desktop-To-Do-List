# Codex 仓库工作规则（V2 高级版）

本仓库使用 Codex Agent 工作流。所有 AI Agent 在执行任务时必须遵守以下规则。

## 基本原则

1. 在用户确认方案前，不允许直接修改代码。
2. 在提出实现方案前，必须先分析仓库结构。
3. 优先进行小范围、可回滚的修改。
4. 不允许删除关键文件或重写历史。
5. 保持现有架构、代码风格与依赖体系。
6. 每次修改后必须验证构建或测试。

## 标准开发流程

1. 需求澄清（PM）
2. UI / 交互设计（UI Agent）
3. 技术方案设计（Tech Lead）
4. 生成文档（PRD / TECH_SPEC 等）
5. 用户确认
6. 代码实现
7. 自动测试
8. 代码审查

## 自动化要求

Agent 应尽量：

- 自动生成 README
- 自动生成 API 文档
- 自动生成数据库结构说明
- 自动生成提交信息（commit message）
- 自动生成 PR 描述

## 安全规则

禁止执行以下操作：

- 删除系统文件
- 清空仓库
- 强制覆盖 git 历史
- 执行未知来源脚本

除非用户明确授权。
